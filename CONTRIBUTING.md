# Contributing to Add4x

Thank you for your interest in contributing to the Add4x Restaurant Platform.

## Getting Started

1. Fork the relevant repository
2. Clone your fork locally
3. Create a feature branch from the repository's primary working branch
4. Make your changes
5. Submit a pull request

## Development Setup

See individual service READMEs for setup instructions:

- [restaurant-backend](https://github.com/Add4x/restaurant-backend)
- [restaurant-admin](https://github.com/Add4x/restaurant-admin)
- [restaurant-ui](https://github.com/Add4x/restaurant-ui)
- [restaurant-kds](https://github.com/Add4x/restaurant-kds)
- [restaurant-pos](https://github.com/Add4x/restaurant-pos)

## Issue Guidelines

- Use the provided issue templates (bug report, feature request, task)
- Search existing issues before creating a new one
- Provide as much detail as possible

## Pull Request Guidelines

- Fill out the PR template completely
- Keep PRs focused — one feature/fix per PR
- Write descriptive commit messages
- Ensure all tests pass before requesting review
- Link related issues in the PR description

## Code Standards

- **Node.js 20 LTS** for all services
- **pnpm** for package management
- **TypeScript** for type safety
- Follow existing code style and conventions in each repo
- Write tests for new functionality

## Branch Naming

Use descriptive branch names with a prefix:

- `feat/` — new features
- `fix/` — bug fixes
- `chore/` — maintenance tasks
- `docs/` — documentation changes
- `refactor/` — code restructuring

## Labels

Issues and PRs use a standardized label system across all repos. Labels are managed centrally and synced automatically. See `.github/labels.yml` for the full list.

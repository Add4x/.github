# Support

## Before Opening an Issue

- Check the relevant service README for setup and troubleshooting steps.
- Search existing issues before filing a new one.
- Use the provided issue forms so maintainers get the details needed to act quickly.

## Where to Ask for Help

- Product or implementation bugs: open an issue in the relevant repository.
- Security concerns: follow the guidance in `SECURITY.md` and do not open a public issue.
- Cross-service planning or coordination questions: use the platform tracker project once configured.

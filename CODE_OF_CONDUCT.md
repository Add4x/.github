# Code of Conduct

## Our Standard

We expect contributors to act professionally, communicate clearly, and keep discussions focused
on improving the Add4x platform.

Examples of positive behavior include:

- giving direct, evidence-based technical feedback
- assuming good intent while challenging weak reasoning
- documenting decisions and tradeoffs clearly
- respecting maintainers' time with reproducible reports and scoped pull requests

Examples of unacceptable behavior include:

- harassment, personal attacks, or discriminatory language
- intentionally disruptive behavior
- sharing private customer, employee, or security-sensitive information publicly

## Enforcement

Project maintainers may remove comments, close discussions, or restrict participation when this
code of conduct is violated.

## Reporting

For conduct concerns, contact the repository maintainers privately instead of opening a public issue.
